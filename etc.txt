Изучение Git для новичков

1 git init - создает GIT репозиторий
2 git branch 'name' - создание ветки
3 git branch -a - показывает все созданные ветки
4 git status - проверить все изменения
5 git add '' - добавить файлы в локальный репозиторий
6 git checkout 'id or branch' - вернуться до определенного коммита по ID или названия ветки
7 git checkout master - вернуться на основную ветку
8 git checkout -b 'name' - создать новую ветку и сразу перейти на нее
9 git commit -m "'comment'" - добавить изменения в ветку с определенным комментарием
10 git merge 'name' - объединить основную (master) ветку с 'name'
11 git log - просмотреть все коммиты. Выход через q
12 git log --oneline - просмотреть все коммиты в линию
13 git revert 'ID' - возврат проекта к определенному по ID коммиту
14 git reset 'ID' - удаляет все коммиты начиная с ID
15 git reset 'ID' --hard - удаляет все коммиты начиная с ID и возвращает все файлы до данного ID
16 git remote add origin 'name' - соединяет с GitHub репозиторием
17 git push -u origin 'name' - загружает ветку в GitHub репозиторий
18 git clone 'address' - копирует весь репозиторий
19 git pull - добавляет последнюю версию проекта в папку

git log --pretty=format:"%h - %s : %ad [ %an ]" - показывает все коммиты в определенном формате: "ID - название : дата создания [автор]"

. - все файлы
*.ext - файлы с определенным расширением
name/ - файлы в определенной папке
name/**/ - файлы в подпапках папки name

В файл с расширением .gitignore прописываются через новую строку все игнорированные для GIT файлы
